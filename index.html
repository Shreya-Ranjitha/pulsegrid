FAQ

Common questions.

What data formats does Pulsegrid support?

Pulsegrid natively ingests GTFS, GTFS-RT, SIRI, and raw NMEA GPS streams. For custom formats, the integration layer supports any REST, MQTT, or WebSocket source via a no-code connector builder.

How does the 12ms ingest latency work in practice?

We maintain persistent event-driven connections to your data sources rather than polling on intervals. The 12ms figure is median end-to-end: from the vehicle's GPS unit emitting a coordinate to that position appearing in the Pulsegrid API and dashboard.

Can I migrate from an existing AVL system?

Yes. Pulsegrid can run alongside your existing CAD/AVL system during a parallel period. Our onboarding team handles the connector setup. Enterprise customers get a dedicated migration engineer.

What's included in the 99.98% uptime SLA?

The SLA covers platform API availability, real-time data ingestion pipeline, and dashboard uptime. It includes credits for any breach, a 24/7 incident response team, and a public status page with full historical uptime data.
